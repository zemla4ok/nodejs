Semver

Semantic versioning

Semver

  • semver.org
  • Set of rules
  • MAJOR.MINOR.PATCH / X.Y.Z

Semver - Rules

  • Declare public API
  • X, Y, Z - int numbers
  • Restricted to change published versions
  • any changes - new version

Semver - MAJOR

  • backward incompatible changes of API
  • 0 - initial version

Semver - MAJOR

  • SHOULD be increased after making of backward INcompatible changes
  • CAN includes changes for minor and patch versions
  • minor and patch versions SHOULD be set to 0 when increasing major version

Semver - MINOR

  • adding new functionality or updating already implemented without broking backward compatibility
  • SHOULD be increased after making of backward compatible changes
  • patch versions SHOULD be set to 0 when increasing minor version

Semver - PATCH

  • making of backward compatible changes
  • SHOULD be increased after making of backward compatible bug-fixes

npm

Node package manager

npm

  • 850+ thousand of packages
  • docs.npmjs.com
  • github.com/npm/npm

npm - package

  • a) directory with program + package.json
  • b) gzip archive of (a)
  • c) reference to (b)
  • d) name@version in registry of (c)
  • e) name@tag to (d)
  • f) name@latest to (d)
  • g) link to git-repo with (a)
